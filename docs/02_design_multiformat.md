# マルチフォーマットコンテンツ変換プラットフォーム 設計書・要件定義書

**プロジェクト名（仮）:** Transmemo / Formify / Memoshift  
**バージョン:** 0.1  
**作成日:** 2026-02-28

---

## 1. プロジェクト概要

### 1.1. コンセプト

> **「あなたの記憶を、あなたの好きな形に。」**

写真・テキスト・URLなど、あらゆる「素材」を入力として受け取り、AIが**音楽・小説・ノベルゲーム・動画・マンガ**など、ユーザーが選んだ好みのメディア形式に自動変換して出力するプラットフォーム。生成したコンテンツはコミュニティで共有・発見できる。

学習・暗記という文脈から出発しつつ、「情報をどう消化するか」という問いに対してユーザーが自分の認知スタイルに合った答えを選べる設計とする。テキストを読むより歌で覚えたい人、物語として体験したい人、ビジュアルノベルで感情を動かして記憶したい人——それぞれの「最適な形」を提供する。

### 1.2. 背景と課題

既存の学習支援ツールは、「テキスト→フラッシュカード」「テキスト→要約」といった単一の変換パスしか持たない。しかし人間の記憶・学習スタイルは多様であり（視覚優位・聴覚優位・物語型・体験型など）、一つの形式がすべての人に最適とは言えない。また、Utamemoのような先行サービスは音楽生成に特化しているが、コミュニティ機能が弱く、ユーザーが作ったコンテンツが循環しない。

本プラットフォームは、**インプットの柔軟性**と**アウトプットの多様性**、そして**コミュニティによる知識の循環**を三本柱として設計する。

## 2. インプット形式

ユーザーが素材として投入できる入力ソースは以下の通りである。変換エンジンは入力形式を問わず、まずAIが内容を解析・構造化した上で各アウトプット形式に変換する。

| インプット種別 | 具体例 | 処理方法 |
| :--- | :--- | :--- |
| **テキスト直接入力** | 授業ノート、メモ、箇条書き、文章 | そのままLLMへ投入 |
| **画像・写真** | 教科書のページ、ホワイトボード、手書きノート | OCR + Vision AIで内容抽出 |
| **PDF / ドキュメント** | 参考書、資料、論文 | テキスト抽出後にLLMへ投入 |
| **WebページURL** | ニュース記事、Wikipedia、ブログ | スクレイピング + 要約 |
| **音声・動画** | 講義録音、YouTube動画 | 音声認識（Whisper等）でテキスト化後に変換 |

---

## 3. アウトプット形式（変換モード）

ユーザーは入力後に「どの形式に変換するか」を選択する。各モードはAIパイプラインが異なり、必要なAPIも異なる。

| モード名 | 概要 | 使用AI/技術 | 認知スタイル適合 |
| :--- | :--- | :--- | :--- |
| **🎵 Song（暗記ソング）** | 内容を歌詞に変換し、AIが楽曲を生成する | Gemini（歌詞）+ Mureka AI（楽曲） | 聴覚優位、リズム記憶 |
| **📖 Novel（短編小説）** | 内容を物語・ストーリーとして再構成する | Gemini / Claude（長文生成） | 物語型、感情記憶 |
| **🎮 Visual Novel（ノベルゲーム）** | キャラクターが登場する対話形式のノベルゲームシナリオを生成し、ブラウザ上でプレイできる | Gemini（シナリオ）+ 画像生成AI（背景・立ち絵）| 体験型、感情記憶 |
| **🎬 Video（解説動画スクリプト）** | 内容をYouTube風の解説動画スクリプトに変換し、将来的にはAI動画生成と連携 | Gemini（スクリプト）+ Sora / Runway（動画） | 視覚優位 |
| **🗺️ Mind Map（マインドマップ）** | 内容を階層構造のマインドマップとして可視化する | Gemini（構造化）+ D3.js / Mermaid（描画） | 構造型、空間記憶 |
| **🃏 Flashcard（フラッシュカード）** | 重要語句をQ&A形式のカードに変換する（Anki互換） | Gemini（Q&A抽出） | 反復学習型 |
| **🎙️ Podcast（対話形式音声）** | 二人のホストが議論する形式のポッドキャスト台本を生成し、TTS音声で出力 | Gemini（台本）+ TTS（音声合成） | 聴覚優位、対話型 |

### 3.1. MVP（最初のリリース）で実装するモード

全モードを一度に実装するのはリスクが高い。MVPでは以下の3モードに絞り、コア体験を磨くことを優先する。

1. **🎵 Song（暗記ソング）** — 既存のUtamemoに対する明確な差別化軸であり、Mureka APIが使える。
2. **📖 Novel（短編小説）** — LLMのみで完結するため実装コストが低く、物語型ユーザーへの訴求力が高い。
3. **🃏 Flashcard（フラッシュカード）** — 最もシンプルな変換で、学習ツールとしての実用性が最も高い。

残りのモード（Visual Novel、Video、Mind Map、Podcast）はv2以降でロードマップに組み込む。

## 4. コミュニティ・共有機能

生成したコンテンツは個人ライブラリに保存されるだけでなく、公開設定によってコミュニティで共有・発見できる。これがUtamemoとの最大の差別化点である。

### 4.1. ディスカバリー（発見）

ユーザーは他者が作成したコンテンツを以下の方法で発見できる。

- **タグ検索:** `#日本史` `#英単語` `#TOEIC` などの学習タグで絞り込む。
- **モード別フィルタ:** 「Songだけ見たい」「Novelだけ見たい」といったフィルタリング。
- **ランキング:** いいね数・再生数・保存数によるデイリー/ウィークリーランキング。
- **フィード:** フォローしているユーザーの新着コンテンツ。

### 4.2. インタラクション

| 機能 | 概要 |
| :--- | :--- |
| **いいね** | コンテンツへのワンタップ評価 |
| **コメント** | 感想・学習成果・補足情報の書き込み |
| **ブックマーク** | 他者のコンテンツを自分のライブラリに保存 |
| **リミックス** | 他者のコンテンツを別のモードに変換する（例：SongをNovelに変換） |

**「リミックス」機能**は本プラットフォーム独自の機能であり、「誰かが作った暗記ソングを、自分は小説形式で読みたい」というニーズに応える。コンテンツの多様な消費・再利用を促し、コミュニティの活性化につながる。

## 6. 技術アーキテクチャ

### 6.1. AIパイプライン

各変換モードのAIパイプラインは以下の通りである。すべてのモードで共通する前処理として、入力素材の「内容理解・構造化」をGeminiが担う。

```
[インプット素材]
      ↓
[前処理レイヤー]
  ├─ テキスト → そのまま
  ├─ 画像 → Vision AI (Gemini Vision) でOCR・内容抽出
  ├─ PDF → テキスト抽出
  ├─ URL → スクレイピング + 要約
  └─ 音声/動画 → Whisper で文字起こし
      ↓
[コンテンツ理解・構造化]
  Gemini: 主題・キーワード・構造を抽出
      ↓
[モード別変換]
  Song → Gemini (歌詞) → Mureka AI (楽曲生成)
  Novel → Gemini (物語生成)
  Visual Novel → Gemini (シナリオ) + 画像生成AI (背景・立ち絵)
  Mind Map → Gemini (階層構造JSON) → D3.js描画
  Flashcard → Gemini (Q&A抽出)
  Podcast → Gemini (台本) → TTS (音声合成)
```

### 6.2. 技術スタック（推奨）

| レイヤー | 技術 | 理由 |
| :--- | :--- | :--- |
| **フロントエンド** | React + TypeScript + TailwindCSS | SPA、型安全、スタイリング効率 |
| **バックエンド** | Node.js + tRPC | 型共有、API設計の効率化 |
| **データベース** | MySQL (Drizzle ORM) | リレーショナルデータの管理 |
| **ファイルストレージ** | S3互換ストレージ | 音声・画像・動画ファイルの保管 |
| **AI: 内容理解・テキスト生成** | Gemini API (Google) | 長文処理、Vision対応、日本語品質 |
| **AI: 楽曲生成** | Mureka AI API | Song変換モード専用 |
| **AI: 画像生成** | Stable Diffusion / DALL-E | Visual Novel変換モード用 |
| **AI: 音声合成** | Google TTS / ElevenLabs | Podcast変換モード用 |
| **認証** | Manus OAuth | ユーザー管理 |

## 10. 付加価値の整理

本プラットフォームが既存サービスに対して持つ本質的な差別化は、以下の3点に集約される。

**① 認知スタイルの自由化**  
「どうやって覚えるか」をユーザーが選べる。テキストを読むのが苦手な人は歌で、物語が好きな人は小説で、ゲームが好きな人はノベルゲームで——同じ情報を異なる形式で消費できることで、より多くの人に「刺さる」体験を提供できる。

**② リミックスによる知識の循環**  
他者のコンテンツを別形式に変換する「リミックス」機能により、一つのコンテンツが複数の形式で派生・流通する。これはUGC（ユーザー生成コンテンツ）プラットフォームとしての独自性であり、TikTokのデュエット機能やYouTubeのカバー文化に近い概念を学習領域に持ち込む試みである。

**③ 学習コンテキストの保持**  
単なるエンタメコンテンツ生成ツールではなく、「何を覚えるためのコンテンツか」というメタ情報（タグ・科目・資格名）を保持・検索できる設計とすることで、学習プラットフォームとしての実用性を担保する。
