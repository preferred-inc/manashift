# 暗記ソング共有アプリ「MelodyMind」設計書・要件定義書

## 1. 概要

本ドキュメントは、AI音楽生成技術を活用した学習支援アプリ「MelodyMind」の設計および要件を定義するものである。

### 1.1. プロジェクトの背景と目的

記憶・暗記という学習における普遍的な課題に対し、音楽の持つ記憶定着効果とAIによるパーソナライズされたコンテンツ生成能力を組み合わせることで、全く新しい学習体験を提供することを目的とする。特に、ユーザーが生成した「暗記ソング」を共有し、学び合う「コミュニティ」の側面を重視することで、学習の継続性を高め、ユーザーエンゲージメントの最大化を図る。

先行サービスである「Utamemo」は、テキストや画像から楽曲を生成する基本機能を提供するが [1]、本プロジェクトでは、これをさらに発展させ、ユーザー間のインタラクションを活発化させることで、単なるツールに留まらない学習プラットフォームの構築を目指す。

### 1.2. コンセプト

**「暗記を、みんなの歌声で。」**

退屈な暗記作業を、AIによる作曲で楽しく効率的な学習体験に変えるだけでなく、他の学習者が作った歌で学び、自分の歌で誰かの学びを助ける、知識の循環が生まれるコミュニティを創造する。

### 1.3. ターゲットユーザー

- **プライマリ:** 資格試験や大学受験を控える中高生、大学生、社会人
- **セカンダリ:** 語学学習者、子供に楽しく勉強させたい親、新しい知識の習得に意欲的な全ての人

## 2. 機能要件

### 2.1. 機能一覧

| 大項目 | 中項目 | 機能概要 | 優先度 |
| :--- | :--- | :--- | :--- |
| **コア機能** | **楽曲生成** | テキスト（手入力、URL、ファイル読込）を基に、AIが歌詞・楽曲を自動生成する。 | 高 |
| | **ライブラリ** | 生成・保存した楽曲を管理・再生する個人スペース。 | 高 |
| **コミュニティ機能** | **コンテンツ共有** | 生成した楽曲を公開し、他のユーザーと共有する。 | 高 |
| | **発見（Discovery）** | 他のユーザーが作成した楽曲を検索・閲覧する。 | 高 |
| | **インタラクション** | 楽曲に対する「いいね」、コメント、ブックマーク（ライブラリ追加）機能。 | 高 |
| | **ユーザープロフィール** | 各ユーザーの活動（作成した楽曲、いいねした楽曲等）を表示するページ。 | 中 |
| | **フォロー** | 他のユーザーをフォローし、活動をフィードで追跡する。 | 中 |
| **学習支援機能** | **クイズ生成** | 楽曲の歌詞から、自動で穴埋め問題などのクイズを生成する。 | 中 |
| | **プレイリスト** | 複数の楽曲をテーマ別にまとめることができる。 | 中 |
| **その他** | **ユーザー認証** | 新規登録、ログイン、退会機能。 | 高 |
| | **外部連携** | 生成した楽曲をSNS（X, TikTok等）にシェアする機能。 | 低 |

### 2.2. 機能詳細

#### 2.2.1. 楽曲生成

- **入力ソース:** テキストボックスへの直接入力、WebページのURL指定、テキストファイル（.txt, .md）やPDFファイルのアップロードに対応する。
- **AIモデル:**
    - 歌詞生成・要約: Gemini APIを活用し、長文からの要点抽出や歌詞形式への変換を行う。
    - 楽曲・歌声生成: Mureka AI APIを利用する [2]。
- **カスタマイズ項目:** ユーザーは以下の項目を選択できる。
    - ジャンル（ポップ、ロック、ジャズ、ヒップホップ等）
    - ボーカル（男性、女性、中性）
    - テンポ（遅い、普通、速い）

#### 2.2.2. コミュニティ機能

- **楽曲の公開設定:** 楽曲生成後、「公開」「非公開」を選択できる。
- **発見（Discovery）機能:**
    - **検索:** キーワード（楽曲名、タグ、歌詞の一部）による検索。
    - **カテゴリ:** 「#日本史」「#英単語」のようなタグによる絞り込み。
    - **ランキング:** 「いいね」数、再生数、コメント数によるソート（デイリー、ウィークリー、マンスリー）。
    - **フィード:** フォローしているユーザーの新着楽曲を表示するタイムライン。
- **インタラクション:**
    - **いいね:** ワンタップで楽曲を評価できる。
    - **コメント:** 楽曲に対する感想や学習成果などを書き込める。
    - **ブックマーク:** 気に入った楽曲を自身のライブラリに保存し、いつでも聴けるようにする。

## 3. 非機能要件

### 3.1. UX/UI設計

- **デザインコンセプト:** 学習アプリとしての信頼性と、音楽アプリとしての楽しさを両立させる。直感的でシンプルな操作性を最優先する。
- **コミュニティの活性化:** SoundCloudのような成功事例を参考に [3]、ユーザーが自然とコンテンツを投稿し、他者と交流したくなるような動線設計を行う。

### 3.2. 技術要件

- **フロントエンド:** Webブラウザ（PC/スマートフォン）で動作するシングルページアプリケーション（SPA）として開発する。将来的なアプリ化を視野に入れ、ReactやVue.jsなどのフレームワークを採用する。
- **バックエンド:** Mureka AI API、Gemini APIと連携。データベースやユーザー認証は、FirebaseやSupabaseのようなBaaS (Backend as a Service) を活用し、開発速度を向上させる。
- **インフラ:** Render, Vercel, AWS, Google Cloudなどのスケーラブルなクラウドプラットフォームを利用する。

### 3.3. マネタイズ

- **フリーミアムモデル** を採用する。
    - **無料プラン:**
        - 楽曲生成数/日に制限（例: 3曲/日）。
        - ライブラリ保存曲数に上限（例: 10曲）。
        - アプリ内に広告を表示。
    - **有料プラン（サブスクリプション）:**
        - 楽曲生成、保存が無制限。
        - 広告の非表示。
        - 高度な楽曲カスタマイズ機能（声質や楽器の指定など）の解放。
        - 楽曲の音声ファイル（MP3）ダウンロード機能。
